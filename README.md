AI Visual Studio Extension
DCA AI is a Visual Studio 2022 extension that allows developers to query internal or local AI models directly from the IDE. It supports multiple connection modes (WebSocket, REST, OpenAI-compatible, Open WebUI) and can act as an MCP (Model Context Protocol) server.

Features
Multi-Mode Support: Connect to various AI backends including:
WebSocket streams
OpenAI / Llama.cpp compatible endpoints
Open WebUI instances
Generic REST APIs
MCP Server Capability: Expose your local AI setup as an MCP server via Streamable HTTP, Stdio, or SSE.
Context-Aware Prompting: Include specific context in your prompts:
Current selection
Active file content
Full project source
Open tabs
VS Compiler errors
Secure Configuration: Secrets (API keys/tokens) are encrypted at rest using Windows DPAPI (CurrentUser scope). The config file is safe to commit to source control.
Diff Approval: Optional mode to review code changes generated by the AI before applying them to your source.
Installation & Setup
Prerequisites
Visual Studio 2022 (17.x)
.NET 8 (or later)
Configuration
Open Visual Studio and ensure a project is active.
Go to the Extensions menu -> Ask Internal AI -> Configuration (or open the config tool window if available).
Adjust the settings in the configuration UI:
Modes: Select which connection protocols to enable.
Connection: Set the Host, Port, and TLS options for your AI server.
Authentication: Choose the auth type (Bearer, API Key, etc.) and enter your secret. The secret is encrypted locally.
Model: Specify the model name (e.g., llama-3-8b).
MCP Server: If you want VS to act as an MCP server, enable transports and set the port.
Click Save to persist settings to .dcaai.json in your project root.
Click Test Connection to verify connectivity.
Usage
Asking the AI
With a project open, go to Extensions -> Ask Internal AI -> Ask Internal AI.
The tool window opens, showing the current server status.
Select Context: Check the boxes for the context you want to include (e.g., "Selection" if you've highlighted code, "Active file" for the current buffer).
Note: The extension calculates the payload size. If it exceeds limits, the text will turn red.
Enter Prompt: Type your question or instruction in the text box.
Send: Click Ask AI.
The response streams into the bottom panel.
If "Require diff approval" is enabled in config, a bar will appear asking you to Accept or Reject the proposed changes.
Acting as an MCP Server
If you enabled MCP Server in the configuration:

The extension listens on the configured port (default: 8090).
It exposes tools/functions via the selected transport (Streamable HTTP, Stdio, or SSE).
You can integrate other tools that support MCP with your VS instance.

Architecture
AIConfiguration: Handles loading/saving the JSON config and encrypting/decrypting secrets using System.Security.Cryptography.ProtectedData.
ToolWindow1Command: The entry point command that captures the active project and editor context.
ConfigToolWindowData: The ViewModel for the configuration UI, handling property binding and persistence.
AiBackendFactory: (Implied) Factory pattern used to create the appropriate backend connector based on the selected mode.
Troubleshooting
"No active project found": Ensure you have a solution or project open in Visual Studio before running the command.
Connection Failed:
Verify the Host and Port in the config match your AI server.
Check if TLS is enabled on the server and match the UseTls setting.
Use the Test Connection button in the config window to debug issues.
Secret Not Found: If you move the .dcaai.json file to another machine, you must re-enter the secret, as DPAPI encryption is machine/user-specific.
